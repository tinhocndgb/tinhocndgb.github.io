Đề kiểm tra GHK I - Tin học 12 - Đề 1

Hệ quản trị cơ sở dữ liệu (DBMS) là gì?

Giải thích: DBMS (Database Management System) là hệ thống phần mềm cho phép tạo, quản lý và khai thác CSDL.

Mô hình CSDL quan hệ (relational database) tổ chức dữ liệu dưới dạng gì?

Giải thích: Mô hình quan hệ tổ chức dữ liệu thành các bảng (tables) với các hàng (records) và cột (fields).

Khóa chính (primary key) trong bảng có đặc điểm gì?

Giải thích: Khóa chính phải có giá trị duy nhất, không NULL, mỗi bảng chỉ có một khóa chính.

SQL là viết tắt của?

Giải thích: SQL = Structured Query Language, ngôn ngữ truy vấn có cấu trúc dùng để thao tác với CSDL.

Câu lệnh SQL nào dùng để truy vấn dữ liệu từ bảng?

Giải thích: SELECT dùng để truy vấn (query) dữ liệu, các lệnh khác dùng để thêm, sửa, xóa.

Mệnh đề WHERE trong SQL dùng để làm gì?

Giải thích: WHERE dùng để chỉ định điều kiện lọc các bản ghi cần truy vấn.

Khóa ngoại (foreign key) là gì?

Giải thích: Khóa ngoại tạo liên kết giữa các bảng, tham chiếu đến khóa chính của bảng khác.

Câu lệnh nào tạo bảng SINHVIEN với các trường MaSV, HoTen, NgaySinh?

Giải thích: CREATE TABLE dùng để tạo bảng mới với các trường và kiểu dữ liệu.

Hàm COUNT() trong SQL dùng để làm gì?

Giải thích: COUNT() đếm số bản ghi, SUM() tính tổng, AVG() tính trung bình, MAX() tìm giá trị lớn nhất.

Thuật ngữ "chuẩn hóa dữ liệu" (normalization) trong CSDL nhằm mục đích gì?

Giải thích: Chuẩn hóa giúp tổ chức CSDL hiệu quả, giảm dư thừa và tránh các dị thường dữ liệu.

Mệnh đề GROUP BY trong SQL dùng để làm gì?

Giải thích: GROUP BY nhóm các bản ghi có cùng giá trị, thường đi kèm với các hàm tập hợp như COUNT, SUM.

Trong thiết kế CSDL, mối quan hệ "nhiều-nhiều" (many-to-many) được giải quyết bằng cách nào?

Giải thích: Mối quan hệ nhiều-nhiều cần bảng trung gian để lưu các cặp khóa từ hai bảng.

Đánh dấu Đúng (Đ) hoặc Sai (S) cho các câu sau:

CSDL (Database) là tập hợp các dữ liệu có liên quan được tổ chức một cách có cấu trúc.
Giải thích: Đúng, đây là định nghĩa cơ bản của CSDL.
Trong mô hình quan hệ, thứ tự các hàng trong bảng là quan trọng.
Giải thích: Sai, trong mô hình quan hệ, thứ tự các hàng không quan trọng.
Câu lệnh DELETE FROM TableName xóa tất cả các bản ghi trong bảng.
Giải thích: Đúng, nếu không có mệnh đề WHERE, DELETE sẽ xóa tất cả bản ghi.
Khóa chính (primary key) có thể có giá trị NULL.
Giải thích: Sai, khóa chính không được phép có giá trị NULL.

Đánh dấu Đúng (Đ) hoặc Sai (S) cho các câu sau:

SQL là ngôn ngữ phân biệt chữ hoa chữ thường.
Giải thích: Sai, SQL không phân biệt chữ hoa chữ thường (case-insensitive).
Mệnh đề ORDER BY dùng để sắp xếp kết quả truy vấn.
Giải thích: Đúng, ORDER BY sắp xếp kết quả theo cột chỉ định.
INNER JOIN chỉ trả về các bản ghi có khớp ở cả hai bảng.
Giải thích: Đúng, INNER JOIN chỉ lấy các bản ghi có giá trị khớp trong cả hai bảng.
TRUNCATE TABLE và DELETE FROM đều có thể rollback (hoàn tác) được.
Giải thích: Sai, TRUNCATE thường không thể rollback, DELETE có thể rollback trong transaction.

Câu lệnh SQL nào dùng để lấy tất cả dữ liệu từ bảng SinhVien?

Giải thích: SELECT * FROM SinhVien - SELECT để truy vấn, * để chọn tất cả cột.

Mệnh đề nào trong SQL dùng để lọc bản ghi có Diem > 5?

Giải thích: WHERE Diem > 5 - WHERE dùng để lọc theo điều kiện.

Kiểu dữ liệu nào trong SQL dùng để lưu số nguyên?

Giải thích: INT (integer) dùng cho số nguyên. Các kiểu khác: FLOAT (số thực), VARCHAR (chuỗi).

Hàm nào đếm tổng số bản ghi trong bảng?

Giải thích: COUNT(*) đếm tất cả bản ghi, COUNT(column) đếm bản ghi không NULL của cột.

Cho CSDL QUANLYHOCSINH với các bảng:
HOCSINH(MaHS, HoTen, NgaySinh, GioiTinh, MaLop)
LOP(MaLop, TenLop, SiSo, MaGVCN)
GIAOVIEN(MaGV, HoTen, MonHoc)
Viết các câu lệnh SQL để:
a) Tìm tất cả học sinh nam
b) Đếm số học sinh trong mỗi lớp
c) Tìm học sinh có ngày sinh sau 2005-01-01

Đáp án Tham khảo:
a) Tìm tất cả học sinh nam:
```sql
SELECT * FROM HOCSINH
WHERE GioiTinh = 'Nam';
-- Hoặc GioiTinh = 'M' tùy cách lưu
```

b) Đếm số học sinh trong mỗi lớp:
```sql
SELECT MaLop, COUNT(*) AS SoHocSinh
FROM HOCSINH
GROUP BY MaLop;
```

c) Tìm học sinh có ngày sinh sau 2005-01-01:
```sql
SELECT * FROM HOCSINH
WHERE NgaySinh > '2005-01-01';
```

d) (Thêm) Tìm thông tin học sinh và tên lớp của họ:
```sql
SELECT HOCSINH.MaHS, HOCSINH.HoTen, LOP.TenLop
FROM HOCSINH
INNER JOIN LOP ON HOCSINH.MaLop = LOP.MaLop;
```

Trình bày sự khác nhau giữa các loại JOIN trong SQL: INNER JOIN, LEFT JOIN, RIGHT JOIN, FULL OUTER JOIN. Cho ví dụ minh họa.

Đáp án Tham khảo:
- INNER JOIN:
  + Chỉ trả về các bản ghi có khớp ở CẢ HAI bảng
  + Ví dụ: Học sinh có điểm (chỉ lấy học sinh có điểm)

- LEFT JOIN (LEFT OUTER JOIN):
  + Trả về TẤT CẢ bản ghi từ bảng bên trái và các bản ghi khớp từ bảng bên phải
  + Nếu không khớp, NULL cho các cột của bảng bên phải
  + Ví dụ: Tất cả học sinh và điểm của họ (kể cả học sinh không có điểm)

- RIGHT JOIN (RIGHT OUTER JOIN):
  + Trả về TẤT CẢ bản ghi từ bảng bên phải và các bản ghi khớp từ bảng bên trái
  + Ngược lại với LEFT JOIN

- FULL OUTER JOIN:
  + Trả về TẤT CẢ bản ghi khi có khớp ở một trong hai bảng
  + Kết hợp của LEFT và RIGHT JOIN
  + Ví dụ: Tất cả học sinh và tất cả lớp (kể cả học sinh không có lớp và lớp không có học sinh)

Ví dụ minh họa:
Giả sử có 2 bảng:
HOCSINH: (1-An, 2-Binh, 3-Chi)
DIEM: (1-8, 2-9) -- Chi không có điểm

```sql
-- INNER JOIN: chỉ An và Binh
SELECT * FROM HOCSINH INNER JOIN DIEM ON HOCSINH.MaHS = DIEM.MaHS;

-- LEFT JOIN: An, Binh, Chi (Chi có NULL cho điểm)
SELECT * FROM HOCSINH LEFT JOIN DIEM ON HOCSINH.MaHS = DIEM.MaHS;
```

Thiết kế CSDL cho hệ thống quản lý thư viện với các yêu cầu:
- Mỗi sách có mã sách, tên sách, tác giả, năm xuất bản, thể loại
- Mỗi độc giả có mã độc giả, họ tên, ngày sinh, địa chỉ
- Mỗi lần mượn ghi nhận mã mượn, mã sách, mã độc giả, ngày mượn, ngày trả dự kiến
a) Xác định các bảng, trường, khóa chính, khóa ngoại
b) Viết câu lệnh SQL tạo các bảng
c) Viết truy vấn tìm sách mượn nhiều nhất

Đáp án Tham khảo:
a) Thiết kế CSDL:
- Bảng SACH(MaSach, TenSach, TacGia, NamXB, TheLoai)
  Khóa chính: MaSach
- Bảng DOCGIA(MaDG, HoTen, NgaySinh, DiaChi)
  Khóa chính: MaDG
- Bảng MUON(MaMuon, MaSach, MaDG, NgayMuon, NgayTraDuKien)
  Khóa chính: MaMuon
  Khóa ngoại: MaSach tham chiếu SACH(MaSach)
  Khóa ngoại: MaDG tham chiếu DOCGIA(MaDG)

b) Câu lệnh SQL tạo bảng:
```sql
CREATE TABLE SACH (
  MaSach INT PRIMARY KEY,
  TenSach VARCHAR(100) NOT NULL,
  TacGia VARCHAR(50),
  NamXB INT,
  TheLoai VARCHAR(30)
);

CREATE TABLE DOCGIA (
  MaDG INT PRIMARY KEY,
  HoTen VARCHAR(50) NOT NULL,
  NgaySinh DATE,
  DiaChi VARCHAR(100)
);

CREATE TABLE MUON (
  MaMuon INT PRIMARY KEY,
  MaSach INT,
  MaDG INT,
  NgayMuon DATE NOT NULL,
  NgayTraDuKien DATE,
  FOREIGN KEY (MaSach) REFERENCES SACH(MaSach),
  FOREIGN KEY (MaDG) REFERENCES DOCGIA(MaDG)
);
```

c) Truy vấn tìm sách mượn nhiều nhất:
```sql
-- Cách 1: Dùng GROUP BY và ORDER BY
SELECT SACH.MaSach, SACH.TenSach, COUNT(*) AS SoLanMuon
FROM MUON
INNER JOIN SACH ON MUON.MaSach = SACH.MaSach
GROUP BY SACH.MaSach, SACH.TenSach
ORDER BY SoLanMuon DESC
LIMIT 1;

-- Cách 2: Dùng subquery
SELECT SACH.MaSach, SACH.TenSach, COUNT(*) AS SoLanMuon
FROM MUON
INNER JOIN SACH ON MUON.MaSach = SACH.MaSach
GROUP BY SACH.MaSach, SACH.TenSach
HAVING COUNT(*) = (
  SELECT MAX(SoLanMuon)
  FROM (
    SELECT COUNT(*) AS SoLanMuon
    FROM MUON
    GROUP BY MaSach
  ) AS temp
);
```
